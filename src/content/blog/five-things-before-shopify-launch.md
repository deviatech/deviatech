Every Launch Track project ends with the same short checklist before we take the password page off. Most of it isn't code — it's the boring operational stuff that actually loses sales if it's missing.

## 1. Payment and COD are both tested with a real order

Not a test order in admin — an actual checkout, on a phone, on mobile data, the way a real customer will do it.

## 2. Product photos are consistent

A store with three different photo styles reads as unfinished, even if the code is perfect. If you don't have a photographer yet, even consistent lighting and a plain background goes a long way.

## 3. Shipping and return policy pages exist

Buyers in Lahore already hesitate to pay online. A missing returns policy is one more reason to abandon the cart.

## 4. Meta Pixel is firing correctly

Not just installed — actually verified with Meta's Events Manager test tool, on the real checkout flow, before you spend a single rupee on ads.

## 5. Someone is watching WhatsApp/DMs on launch day

The first 48 hours after launch is when most questions come in. If nobody's watching, you lose the exact customers who found you fastest.

None of this is complicated. It's just easy to skip when you're excited to launch — which is exactly why we treat it as a checklist, not a memory test.
